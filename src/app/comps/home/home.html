The Problem Data Binding Solves

Before data binding, developers manually synchronized the DOM with application state: every time a variable changed, you had to find the right DOM element and update it with .innerHTML, .value, or setAttribute(). Angular's data binding system automates this synchronization.

The Angular Template-Component Bridge

Every Angular component has two sides:

  1. Component Class (.ts) — the "Model": JavaScript/TypeScript logic and data (properties, methods)
  2. Template (.html) — the "View": HTML that Angular compiles into a live DOM

Data binding is the syntax that lets these two sides communicate. Angular parses the template and sets up live connections — called bindings — between DOM nodes and component properties.

Change Detection Overview

Angular's change detection system is the engine that powers data binding. It tracks changes to component properties and updates the DOM efficiently. When a property changes, Angular re-evaluates the bindings that depend on it and updates only the affected parts of the DOM.

Angular runs Change Detection after every browser event (click, input, keyup), after HTTP responses, and after timers. It checks if any bound property changed; if it did, it updates only the affected DOM nodes. This is far more efficient than re-rendering the whole page.

The 7 Binding Types — One-Page Overview

Angular provides 7 distinct binding types, each with its own syntax and use case:

  1. String Interpolation — {{ text }}: Embed component data into the DOM as text.
  2. Property Binding — [property]="expression": Bind component data to DOM properties (e.g., [disabled], [src]).
  3. Event Binding — (event)="handler": Listen for DOM events and call component methods.
  4. Two-Way Binding — [(ngModel)]="property": Create a two-way data flow between component properties and form inputs.
  5. Attribute Binding — [attr.attribute]="expression": Bind to HTML attributes that don't have corresponding DOM properties.
  6. Class Binding — [class.class-name]=" expression": Conditionally apply CSS classes based on component state.
  7. Style Binding — [style.style-name]="expression": Dynamically set inline styles based on component data.
